# 釣果記録アプリ 説明書（更新版）

釣りになかなか行けないので（天候事情で船が出ず）  
その時間を利用して、釣果記録アプリを作りました。  
そのアプリの説明書です  

---

## 1. アプリ

本アプリは「黒鯛（チヌ）の落とし込み／ヘチ釣り」の釣果を記録し、あとから傾向を分析するための記録アプリです。

---

## 2. できること（機能一覧）

### 釣果ログの記録（1日単位の釣果記録）

- 日時（DATE & TIME）
- エリア／釣り場／ポイント
- 釣り方（STYLE）
- メイン餌（MAIN BAIT）
- 釣果数（TOTAL）
- **あたり数（HIT COUNT）**　← 🆕 NEW!
- 年無し（50cm over）
- サイズ内訳（40–44 / 45–49 / **30–39 / <30** ※任意）← 🆕 30-39cm、30cm未満を追加！
- メモ
- "その他"自由入力（一部項目でリスト外の入力が可能）

### BAIT BREAKDOWN（餌内訳）

- サブ餌を最大3つまで記録
- メイン餌の数は自動計算表示

### CONDITION（任意）

- 天気／潮／風（ON/OFF）

### DATA（分析）

- 期間・エリア・釣り場で絞り込み
- 月別推移（棒グラフ）← 🆕 6段階のサイズ分類に拡張！
- 餌比率／エリア比率／天気・潮・風（円グラフ）
- CSV出力
- 編集／削除／全削除

### AREA（エリア管理）← 🆕 NEW!

- エリア・場所・ポイントの追加／編集／削除
- 階層構造での管理

### BAIT（餌管理）← 🆕 NEW!

- 餌の追加／編集／削除
- デフォルト餌とカスタム餌の管理

NICKNAMEは一度入れたら次回以降も残ります（あとで変更もOK）。

## 4. 記録ルール（このアプリの前提）

本アプリは 「1日単位の釣果を記録する」 使い方を想定していますが、釣果ごとに入力することももちろん可能です。  
**AREA選択は必須です。**

POINTは必須（後から分析しやすくするため）

## 5. LOG（記録）画面：入力手順

### 5-1. DATE & TIME（必須）

「釣れた時刻」ではなく、その日の釣果ログとして記録します  
例：釣行開始時刻／釣行終了時刻／夜釣りなら日付の基準を仲間内で統一

### 5-2. AREA / LOCATION / POINT（必須）

- AREA：エリアを選択
- LOCATION：釣り場を選択
- POINT（必須）：ポイントを選択
- AREAで「その他エリア」を選ぶと、自由入力で場所を記録できます

その他エリアは自由入力で場所を残せます

### 5-3. STYLE（釣り方）

- リストから選択
- 「その他」を選ぶと自由入力欄が表示されます

STYLEは「その他」で自由入力できます。

### 5-4. MAIN BAIT（メイン餌）

- リストから選択
- 「その他」を選ぶと自由入力欄が表示されます

その他の項目は任意で自由入力できます

### 5-5. 釣果数（TOTAL）

その日の釣果総数を入力します。  
**🆕 ボーズ（釣れなかった日）も記録可能**: 釣果数を `0` にすることで、釣れなかった日も記録できます。DATA タブでは「ボーズ」と表示されます。

### 5-6. あたり数（HIT COUNT）← 🆕 NEW!

**任意項目**: あたりがあった回数を記録できます。

**ルール**:
- あたり数は釣果数以上である必要があります
- 未入力（`---`）でもOK
- 例：あたり10回で5匹釣れた → 掛かり率50%

**使い方**:
```
釣果数 (TOTAL): 5
あたり数 (HIT COUNT): 10
→ あたり10回で5匹釣れた
```

**エラー例**:
```
釣果数 (TOTAL): 10
あたり数 (HIT COUNT): 5
→ ❌ エラー「あたり数は釣果数以上である必要があります」
```

## 6. BAIT BREAKDOWN（餌内訳：必要な時だけ）

- 「詳細入力」をONにすると、サブ餌を最大3つまで記録できます
- メイン餌の数は **TOTAL −（サブ餌合計）** で自動計算され表示されます

普段はOFFでもOK。必要な時だけONにします。  
サブ餌を入れると、メイン餌は自動計算（TOTAL−サブ餌合計）で出ます

---

## 7. SIZE BREAKDOWN（サイズ内訳）← 🆕 拡張！

### 記録できるサイズ

- 年無し（50cm over）← 既存
- **「詳細サイズ(40/45)も記録」をONにすると、以下も記録できます**:
  - 45–49cm ← 既存
  - 40–44cm ← 既存
  - **30–39cm** ← 🆕 NEW!
  - **< 30cm** ← 🆕 NEW!

### サイズ分類の色分け

| サイズ | 色 | 説明 |
|--------|-----|------|
| 年無し (50cm+) | 金色 | 既存 |
| 45-49cm | 赤 | 既存 |
| 40-44cm | 青 | 既存 |
| **30-39cm** | **緑** | 🆕 NEW! |
| **<30cm** | **灰色** | 🆕 NEW! |
| **不明** | **濃灰色** | 🆕 未入力分を自動計算 |

### 不明サイズの自動計算

サイズを入力しなかった釣果は、**「不明」** として自動計算されます。

**例**:
```
釣果数 (TOTAL): 10
年無し (50cm over): 2
45~49cm: 1
40~44cm: 3
→ 不明: 10 - (2+1+3) = 4匹
```

### 注意事項

- 内訳の合計が TOTAL を超える場合は保存できません
- 各サイズの合計 ≤ 釣果数である必要があります

必要な日だけ「詳細サイズON」でサイズを記録できます

内訳合計がTOTALを超えると保存できません

## 10. AREA（エリア管理）← 🆕 NEW!

### できること

- エリア・場所・ポイントの追加
- エリア・場所・ポイントの編集
- エリア・場所・ポイントの削除

### 階層構造

```
エリア名
  ├─ 場所名
  │   ├─ ポイント1
  │   ├─ ポイント2
  │   └─ ポイント3
  └─ 場所名2
      └─ ポイント4
```

### 追加方法

1. AREA タブを開く
2. フォームに入力:
   - AREA NAME: エリア名
   - LOCATION NAME: 場所名
   - POINT NAME: ポイント名
3. 「ADD AREA」ボタンをクリック

### 編集・削除

- **Edit ボタン**: 名前を変更できます
- **Delete ボタン**: 削除できます（確認ダイアログが表示されます）

### 削除の注意事項

- **エリア削除**: そのエリアの全ての場所・ポイントも削除されます
- **場所削除**: その場所の全てのポイントも削除されます
- **ポイント削除**: そのポイントのみ削除されます

削除前に、削除される件数が表示されます。

## 14. バリデーションルール← 🆕 NEW!

入力ミスを防ぐため、以下のチェックが自動で行われます。

### 必須項目

- DATE（日付）
- AREA（エリア）
- LOCATION（場所）

### サイズ内訳チェック

**ルール**: 各サイズの合計 ≤ 釣果数

**例（OK）**:
```
釣果数: 10
年無し: 2, 45-49: 1, 40-44: 3, 30-39: 2, <30: 1
→ 合計 9匹 ≤ 10匹 ✅
```

**例（エラー）**:
```
釣果数: 5
年無し: 2, 45-49: 1, 40-44: 3, 30-39: 1
→ 合計 7匹 > 5匹 ❌
エラー: 「サイズ内訳がトータルを超えています」
```

### あたり数チェック← 🆕 NEW!

**ルール**: あたり数 ≥ 釣果数（あたり数が入力されている場合）

**例（OK）**:
```
釣果数: 5, あたり数: 10 ✅
釣果数: 5, あたり数: 5 ✅（掛かり率100%）
釣果数: 5, あたり数: 未入力 ✅
```

**例（エラー）**:
```
釣果数: 10, あたり数: 5 ❌
エラー: 「あたり数は釣果数以上である必要があります」
```

### 餌別内訳チェック

**ルール**: サブ餌の合計 ≤ 釣果数

メイン餌は自動計算: 釣果数 - サブ餌合計

## 15. よくあるトラブル

### Q1. "その他"を選んでも入力欄が出ない

- 画面を更新（リロード）してください
- それでも改善しない場合は、最新版が反映されていない可能性があります

困ったらまずリロード（最新版が反映されてない時があります）。

### Q2. ログインできない（Invalid login credentials）

- メール／パスワードの入力ミスが多いです
- スマホは末尾スペースが入ることがあるので注意してください

### Q3. サイズ内訳がトータルを超えていますエラー← 🆕 NEW!

- 各サイズの合計が釣果数を超えていないか確認してください
- 例：釣果数5匹なのに、年無し2 + 45-49が1 + 40-44が3 + 30-39が1 = 7匹になっていませんか？

### Q4. あたり数のエラー← 🆕 NEW!

- あたり数は釣果数以上である必要があります
- 例：釣果数10匹なのに、あたり数5回になっていませんか？
- あたりがあっても釣れないことはありますが、釣れたということは必ずあたりがあったということです

## 17. まとめ

このアプリで記録を続けることで、以下のような分析が可能になります:

- **時期別**: いつ釣れているか（月別推移）
- **サイズ別**: どのサイズが多いか（6段階の詳細分類）← 🆕 拡張！
- **餌別**: どの餌が効果的か
- **場所別**: どこで釣れているか
- **条件別**: 天気・潮・風の影響
- **掛かり率**: あたり数と釣果数の比較← 🆕 NEW!
- **ボーズ日**: 釣れなかった日の記録← 🆕 NEW!

記録を続けて、自分なりの釣果パターンを見つけましょう！ 🎣
